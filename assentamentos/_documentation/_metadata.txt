#4) Version Control and Code Adaptation
	- All the files in the rawData folder have the *download date* in the name, which is the last 8 numbers. This was done to make it easier to compare versions of the data provided 
  	  by INCRA, which are undergoing recurrent changes.
	- For this reason, every time you run the r2c or merge with municipalities codes, you need to change the *'reference.data'* parameter in the SETUP step of the codes 
          by putting the same download date as the raw file. Example: Data downloaded on December 23, 2023, 'reference.data' = '20231223' 

#5) Comparing Data
	- The 'crosscheck' folder within the 'code' folder contains the code that compares versions of the settlement data with different download dates. It clarifies which areas
	  are new or unique to each version. The results of this experiment are saved on the 'crosscheck' folder within the 'data' folder.
